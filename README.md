# Personal Finance Manager (PFM)

Система обліку особистих фінансів: рахунки, транзакції, бюджети та фінансові цілі.

---

## Що потрібно перед початком

Потрібна лише одна програма — **Node.js версії 18 або новіший**.

**Як перевірити чи встановлений:**

Відкрий будь-який термінал і введи:
```
node -v
```
Якщо відповідь схожа на `v18.0.0`, `v20.0.0`, `v22.0.0` — все добре, йди далі.

Якщо пише "не знайдено" або "not recognized" — завантаж Node.js з [nodejs.org](https://nodejs.org/), встанови, і перезапусти термінал.

---

## Як відкрити термінал у папці проєкту

1. Відкрий папку з проєктом у **Провіднику (File Explorer)**
2. Клікни на адресний рядок зверху (де показується шлях до папки)
3. Введи `cmd` і натисни **Enter**

Відкриється чорне вікно — це командний рядок вже у потрібній папці.

> Альтернатива: клікни правою кнопкою миші на порожньому місці в папці (не на файлі) → "Відкрити термінал" або "Open in Terminal".

## Перше налаштування (лише один раз)

Виконай ці команди по черзі. Кожну вводиш і натискаєш **Enter**, чекаєш поки завершиться.

### Крок 1 — залежності бекенду

```
cd server
```
```
npm install
```

Чекай поки завершиться (може тривати 1–3 хвилини, буде багато тексту — це нормально).

Після завершення:
```
copy .env.example .env
```

Має з'явитись: `1 file(s) copied.`

### Крок 2 — залежності фронтенду

```
cd ..
```
```
npm install
```

Знову чекай поки завершиться.

Після завершення:
```
copy .env.local.example .env.local
```

Має з'явитись: `1 file(s) copied.`

---

## Запуск (кожного разу)

Потрібно **два окремих термінали**. Відкрий два вікна командного рядка у папці проєкту (як описано вище).

### Термінал 1 — бекенд

```
cd server
```
```
node index.js
```

Має з'явитись:
```
PFM Server running on http://localhost:3001
```
Це вікно не закривай — тут працює сервер.

### Термінал 2 — фронтенд

```
npm run dev
```

Має з'явитись щось схоже на:
```
Local:   http://localhost:8080/
```
Це вікно теж не закривай.

### Відкрий у браузері

**http://localhost:8080**

## Типові проблеми

**`npm` не знайдено / "not recognized"**
→ Node.js не встановлений або термінал треба перезапустити після встановлення.

**`cd server` — папка не знайдена**
→ Термінал відкритий не в тій папці. Переконайся що відкрив термінал саме в папці проєкту (де є `package.json` і папка `server`).

**`copy .env.example .env` — файл не знайдено**
→ Ти ще не зайшов в папку `server` командою `cd server`. Перевір де ти знаходишся.

**`npm install` завершився, але `copy` пише помилку**
→ Перевір чи ти в правильній папці: після `cd server` — виконуй `copy .env.example .env`, після `cd ..` — виконуй `copy .env.local.example .env.local`.

**Порт вже зайнятий: `EADDRINUSE :3001` або `EADDRINUSE :8080`**
→ Сервери вже запущені з минулого разу. Відкрий диспетчер задач → вкладка "Подробиці" → знайди процеси `node.exe` → завершити задачу. Потім запускай знову.

**Сторінка не відкривається в браузері**
→ Перевір що обидва термінали запущені і не показують помилок. Бекенд має бути на 3001, фронтенд на 8080.

## Структура проєкту

```
├── server/          # Node.js + Express бекенд
│   ├── routes/      # API маршрути
│   ├── db/          # SQLite схема та початкові дані
│   └── data/        # База даних pfm.db (створюється автоматично)
├── src/             # React фронтенд
│   ├── pages/       # Сторінки додатку
│   ├── components/  # UI компоненти
│   └── store/       # Стан додатку
```

## Функціонал

- Реєстрація та вхід (JWT авторизація)
- Рахунки (картка, готівка, депозит)
- Транзакції (доходи, витрати, перекази між рахунками)
- Категорії доходів і витрат
- Бюджети по категоріях
- Фінансові цілі з прогресом
- Дашборд з графіками
- Експорт транзакцій у CSV / JSON
- Налаштування профілю
